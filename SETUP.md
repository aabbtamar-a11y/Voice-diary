# יומן קול — הוראות הפעלה

אפליקציית ווב (PWA) אישית, חינמית, שיושבת עם אייקון על מסך הבית באייפון.
כל הקוד רץ במכשיר שלך בלבד — אין שרת, אין תשלום. הקלטות נשמרות בתוך האייפון,
ורק כשאת לוחצת "העלה לדרייב" על הקלטה ספציפית היא נשלחת ל-Google Drive שלך.

יש לבצע פעם אחת 3 שלבים: (1) יצירת מפתח ל-Google Drive, (2) העלאה לאחסון חינמי עם כתובת https,
(3) התקנה על מסך הבית. אחר כך האפליקציה פשוט עובדת.

---

## שלב 1 — יצירת Google Client ID (כדי שכפתור "העלה לדרייב" יעבוד)

1. כנסי ל-https://console.cloud.google.com/ עם חשבון הג'ימייל שלך.
2. צרי פרויקט חדש (למעלה בתפריט "בחר פרויקט" → "פרויקט חדש"), תני לו שם כמו `voice-diary`.
3. בתפריט הצד: **APIs & Services → Library** → חפשי "Google Drive API" → לחצי **Enable**.
4. בתפריט הצד: **APIs & Services → OAuth consent screen**.
   - User Type: **External** → Create.
   - מלאי שם אפליקציה (למשל "יומן קול"), האימייל שלך בשדות הנדרשים → Save and Continue בכל השלבים.
   - במסך "Test users" → **Add users** → הוסיפי את כתובת הג'ימייל שלך. זה חשוב — בלי זה לא תוכלי להתחבר.
5. בתפריט הצד: **APIs & Services → Credentials → Create Credentials → OAuth client ID**.
   - Application type: **Web application**.
   - תחת "Authorized JavaScript origins" תוסיפי את הכתובת שתקבלי בשלב 2 (לדוגמה `https://<username>.github.io`).
   - לחצי Create, ותקבלי **Client ID** שנראה כך: `123456-abc.apps.googleusercontent.com`.
6. פתחי את הקובץ [js/config.js](js/config.js) בפרויקט, והחליפי את השורה:
   ```js
   export const GOOGLE_CLIENT_ID = 'YOUR_CLIENT_ID.apps.googleusercontent.com';
   ```
   עם ה-Client ID שקיבלת.

**הערה:** כל עוד האפליקציה במצב "Testing" (ולא עברה תהליך אימות של גוגל), החיבור לדרייב יפוג
אחרי כשבוע ותצטרכי להתחבר מחדש בלחיצה — זה תקין ולא מזיק, זה מוגבל רק לחשבון שלך.

---

## שלב 2 — אחסון חינמי עם כתובת מאובטחת (GitHub Pages)

הקלטה מהמיקרופון ו"הוספה למסך הבית" דורשות שהאתר ירוץ תחת כתובת מאובטחת (https).
GitHub Pages נותן את זה בחינם.

1. אם אין לך חשבון — הירשמי בחינם ב-https://github.com/signup
2. צרי מאגר (repository) חדש: **New repository** → שם, למשל `voice-diary` → **Public** → Create.
3. העלי אליו את כל הקבצים מהתיקייה הזו (`index.html`, `manifest.webmanifest`, `sw.js`, `css/`, `js/`, `icons/`) —
   הכי קל: לגררי את כל הקבצים לתוך העמוד של המאגר בדפדפן (github.com/<username>/voice-diary → Add file → Upload files),
   או דרך Git אם נוח לך יותר.
4. במאגר: **Settings → Pages** → תחת "Build and deployment" → Source: **Deploy from a branch** → Branch: `main`, תיקייה `/ (root)` → Save.
5. אחרי דקה-שתיים תופיע כתובת כמו: `https://<username>.github.io/voice-diary/` — זו הכתובת של האפליקציה שלך.
6. חיזרי לשלב 1.5 בקונסולת גוגל ווודאי שהכתובת הזו מוגדרת תחת "Authorized JavaScript origins".

---

## שלב 3 — התקנה על מסך הבית באייפון

1. פתחי את הכתובת מ-GitHub Pages ב-**Safari** באייפון (חובה Safari, לא כרום).
2. לחצי על כפתור השיתוף (הריבוע עם החץ למעלה) בתחתית המסך.
3. גללי ובחרי **"הוסף למסך הבית" / "Add to Home Screen"**.
4. יופיע אייקון של האפליקציה על מסך הבית — לחיצה עליו פותחת אותה במסך מלא, בלי סרגלי דפדפן.

---

## שימוש יומיומי

- **הקלטה**: לוחצות על כפתור ההקלטה האדום → רואות טיימר רץ → אפשר להשהות (Pause) או לעצור (Stop).
  אחרי עצירה אפשר להוסיף כותרת/הערה קצרה.
- **לוח שנה**: מציג חודש או שבוע, כל יום צבוע ירוק (5+ דקות), צהוב (פחות מ-5), אדום (כלום).
  לחיצה על יום מציגה את כל ההקלטות של אותו יום.
- **רשימה**: כל ההקלטות מכל הימים, אפשר להאזין, לערוך הערה, ולהעלות כל הקלטה בנפרד לדרייב.
- **בדרייב**: ההקלטות עולות לתיקייה בשם "הקלטות יומיות" בדרייב שלך — משם אפשר להאזין או להוריד מהמחשב.

## מגבלות שכדאי לדעת

- ההקלטות נשמרות בתוך האייפון (IndexedDB). אפליקציות שנוספו למסך הבית לא נמחקות
  אוטומטית אחרי 7 ימים כמו גלישה רגילה ב-Safari — אבל אם מוחקים את Safari/מאתחלים את המכשיר,
  ההקלטות שלא הועלו לדרייב יאבדו. מומלץ להעלות הקלטות חשובות לדרייב מדי פעם.
- אם מתקינות את האפליקציה מחדש (לדוגמה במכשיר אחר), היא מתחילה עם היסטוריה ריקה —
  ה"מקור אמת" היחיד המשותף בין מכשירים הוא מה שהעליתן ידנית לדרייב.
